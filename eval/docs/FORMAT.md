# Формат документов и ground truth (для заполнения вручную)

## `docNN.txt`
Обычный текст документа. Абзацы разделяются пустой строкой (`\n\n`) — это
единственное, на что ориентируется нумерация. Чтобы увидеть, какие номера
абзацев увидит модель и под какими номерами их нужно указывать в
ground_truth.json, прогоните:

```
python show_paragraphs.py docs/docNN.txt
```

Это распечатает `[P1] ...`, `[P2] ...` и так далее — именно эти номера
используются как `paragraph` в ground truth.

## `docNN.ground_truth.json`
```json
{
  "doc_id": "docNN",
  "status": "draft",
  "errors": [
    {"type": "factual", "paragraph": 2, "note": "Эверест назван самой глубокой впадиной"}
  ]
}
```

- `status`: `"draft"` — документ ещё не готов, `run_eval.py` его пропустит
  и выведет предупреждение. Когда дозаполнили — поставьте `"ready"`.
- `errors[].type`: один из пяти типов, закрытый список, ровно так:
  `typo`, `grammar`, `factual`, `contradiction`, `logic`.
- `errors[].paragraph`: номер абзаца из `show_paragraphs.py` (целое число,
  без `P` и скобок).
- `errors[].note`: свободный текст для себя, в автоматическом сравнении
  НЕ участвует (сравнение идёт только по паре `type` + `paragraph`).

Если в документе действительно нет ошибок одного из типов — это нормально,
просто не добавляйте записи такого типа. Если в документе нет ошибок вообще —
оставьте `"errors": []`, но `status` всё равно переключите на `"ready"`,
иначе документ останется пропущенным.
